# PDFlab v2 设计:学生 PDF 查阅体验(即时翻译 + 阅读舒适度)

> 状态:**idea 草稿,未立项**(2026-07-06 头脑风暴产物,随时可能推翻或改向)。若确定要做,先复核本文档再出实现计划;确定立项时再纳入 git。
> 背景:v1(product-spec v1.3)已发布。用户重新定位核心需求——"方便用户查阅 PDF",目标用户聚焦**学生**(看文档、看 PDF 书场景多),可能是多语言对照查看,也可能只是单 PDF 查看。v2 主战场是**查看模块**;翻译模块不动。

## 0. 范围一句话

查看模块从"对照阅读器"升级为"学生查阅工具":**即时翻译四形态**(划选气泡 / 双击取词 / 侧栏段译 / 整页即时翻译)+ **扫描页按页 OCR(段落级点选)** + **阅读舒适度四项**(护眼渲染 / 阅读状态记忆 / 页面布局模式 / 专注模式)。

两条工作流互不依赖,可并行:
- **A 线 即时翻译栈**(大头):§1–§4
- **B 线 阅读舒适度**(全是小活):§5

## 1. A 线核心:ViewerTranslationService(Core)

四种翻译形态共用一个服务:

- 包住 `AppState.makeEngine()` 产出的引擎实例,**跟随全局引擎设置**,不新增独立设置项。
- **内置翻译缓存**:key = (文本哈希 + 翻译方向 + 引擎 id),LRU 上限约 500 条,进程内存级(不持久化)。同一段反复点选、翻页来回不重复请求——对免 Key 引擎(Google/DeepL/有道)的限速与封禁保护尤其重要。
- 方向判定复用 `LanguageDetector`,对选中文本即时检测;检测非中英时按现有语义提示。
- 错误全部复用 `PDFLabError` → 现有 L10n 错误文案,在气泡/侧栏内联展示,引擎类错误附"建议切换引擎"。**不新增错误 case**。

## 2. 划选气泡 + 双击取词(App)

- **划选气泡**:监听 `PDFView` 选区变化通知,选区稳定(鼠标松开)后在选区矩形旁弹 `NSPopover`:上原文、下译文,加载中转圈,失败显示错误文案。md/txt 查看侧(NSTextView 选区)顺带支持。
- **双击取词**:双击单词走**系统词典**(`showDefinition(for:at:)`),离线、免费、不占翻译引擎额度,零自研词典。与划选气泡不冲突:双击=词典释义,划选=引擎翻译。

## 3. 段落命中 + 译文侧栏(Core + App)

- Core 新增 **`ParagraphHitTester`**(纯逻辑,TDD):输入页内归一化点击坐标 + 该页段落列表(带 bbox),输出命中段落(或 nil)。段落来源:
  - 文字层页:现有 `PDFTextExtractor` 行提取 + `ParagraphBuilder` 聚段;
  - 扫描页:`PageOCRStore`(§4)。
- **译文侧栏**为右侧面板,两档模式:
  1. **点选段译**:点击正文某段 → 覆盖层画圆角高亮框整段高亮 → 译文追加进侧栏,可累积、可清空;
  2. **整页即时翻译**:开关打开后,当前页全部段落自动批量翻译,侧栏满页对照,翻页自动跟进(即规格 v1 第二节"边看边译"的落地形态)。
- **约束**:译文侧栏与"第二文档对照"共用右侧空间——**仅单文档模式可用**;已打开两个文档时该功能隐藏(对照模式本身已有译文)。

## 4. 扫描页按页 OCR(Core + App)

- Core 新增 **`PageOCRStore`**:按 (文档, 页码) 缓存 OCR 结果。页面首次需要交互(点击/开启整页翻译)时触发:`PageRasterizer`(350 DPI)→ `OCRService`(含语言预嗅探与低置信度预处理重试,全部现成)→ `ParagraphBuilder` 聚段 → 带归一化 bbox 的段落列表入缓存。
- 交互粒度:**段落级点选**——点击扫描页命中 OCR 段落,高亮 + 段译/气泡照常;整页翻译照常。**不做字符级自绘划选**(列为 v2.x 升级项,见 §7)。
- 低置信度页沿用现有语义:侧栏对应译文条目挂"识别质量低"提示。
- OCR 在后台线程执行,页面给轻量"识别中"指示;整页无结果提示"未识别到文字"(现有 L10n key)。

## 5. B 线:阅读舒适度(App 为主)

| 项 | 做法 | 难度 |
|---|---|---|
| 页面布局模式 | PDFKit 原生 `displayMode`:单页 / 双页 / 连续滚动,工具栏分段控件 | 低 |
| 阅读状态记忆 | Core 新增 `ReadingStateStore`(UserDefaults,按文件路径存 页码+缩放倍率+布局模式;上限 50 条、90 天过期清理,风格照抄 `HistoryStore`);重新打开自动恢复 | 低 |
| 专注模式 | 快捷键/工具栏按钮隐藏工具栏+标签条只剩正文,Esc 退出;配合系统全屏 | 低 |
| 护眼渲染 | PDFView 图层挂 CIFilter:夜间=反色+色相旋转 180°("智能反色",图片观感基本正常);羊皮纸=暖色叠加。三档:正常 / 羊皮纸 / 夜间 | 中 |

- 对照模式下布局/护眼对两栏同时生效;阅读状态主副文档各记各的。
- **B 线唯一技术风险**:CIFilter 在 PDFView 图层上的实际表现需真机验证,实现计划中应作为前置 spike。

## 6. 错误处理与测试

- 错误全部走既有 `PDFLabError` → L10n 映射;新增用户可见文案一律进 `L10n`(中英双语),禁止硬编码。
- **TDD 覆盖 Core 新增件**:`TranslationCache`(LRU/去重/key 语义)、`ParagraphHitTester`(命中/边界/空列表)、`ReadingStateStore`(存取/上限/过期)、`PageOCRStore`(缓存命中与触发逻辑;OCR 本体已有测试)。
- UI 层(气泡/侧栏/覆盖层/滤镜/专注模式)靠 `make run` 人工验收;`验收清单.md` 补一节 v2 条目。
- 硬规则照旧:macOS 26/15 双路径 `#available`、`make` 构建、Keychain、conventional commits。

## 7. 非目标(明确不做)

- 扫描页**字符级**自绘划选(v2.x 升级项,段落级接口已留好);
- 批注/高亮**持久化**(打破查看模块只读边界,另议);
- 生词本/学习记录;
- 中英之外的语言;
- 翻译模块的任何改动。

## 8. 实现排序建议(供 writing-plans 参考)

1. **B 线四项**先行(小活,立刻有感;护眼渲染先做 spike);
2. `ViewerTranslationService` + 划选气泡 + 双击取词;
3. `ParagraphHitTester` + 点选段译侧栏;
4. 整页即时翻译(侧栏批量档);
5. `PageOCRStore` 扫描页支持。

每步交付后应用都完整可用。

## 9. 决策记录(为什么这么定)

- **定位转向**:用户明确 v2 围绕"学生查阅 PDF"核心需求,翻译导出退居既有能力;
- **四形态全做**:用户选择一次性全部设计,实现分步;
- **扫描件走查看器内按页 OCR**(而非"提示先用翻译模块处理"):体验完整优先,用户拍板;
- **段落级点选**(而非字符级划选):工程量减半、覆盖约 90% 阅读场景,用户采纳推荐;
- **双击取词用系统词典**:零成本、离线,不烧引擎额度;
- **翻译缓存进程内不持久化**:自用工具,简单优先(符合项目"简单学习项目"定位)。
